# Flow Code

Static analysis graph generation for Python and TypeScript codebases.

**`flowcode`** produces an **execution IR** — a call graph of functions as nodes and calls/contains as edges — that captures the shape of a program in terms of **use cases and execution order** rather than file trees. It is designed to be consumed as a library or via CLI, and is the graph generation layer used by downstream tools.

## Quick start

```bash
cd packages/flowcode
uv venv
uv pip install -e ".[dev,ts]"
uv run python -m pytest tests/ -v
```

```python
from flowcode import generate_graph

graph = generate_graph("/path/to/repo")
print(len(graph["nodes"]), "functions")
print(graph["entrypoints"])
print(graph["use_cases"])
```

CLI:

```bash
flowcode index /path/to/repo -o raw.json
flowcode execution-ir /path/to/repo -o ir.json
flowcode diff before.json after.json
flowcode orphans raw.json overlay.json
flowcode overlay-migrate before.json after.json overlay.json -o new-overlay.json
```

See [`packages/flowcode/README.md`](packages/flowcode/README.md) for the full API, schema, `.flowcode.toml` config, and entrypoint detection cascade.

## Design

Two-artifact split:

- **RAW** — deterministic index from code (files, symbols, refs, calls). Stable technical IDs.
- **Overlay** — friendly names + descriptions + grouping, keyed by RAW IDs. Auto-generated by `flowcode.auto_overlay` (structural naming, optionally enriched with Claude Haiku if `ANTHROPIC_API_KEY` is set).
