# Planning documentation (AI development platform)

This folder holds **structured brainstorming** for a product that helps people **develop other projects** with AI. There is **no demo or prototype code** in this phase—only documents.

**→ Consolidated “where we landed” (read this to avoid getting lost):** [`../product-vision/SPEC.md`](../product-vision/SPEC.md) · **diagram:** [`../product-vision/ARCHITECTURE.md`](../product-vision/ARCHITECTURE.md) · **build order:** [`../product-vision/ROADMAP.md`](../product-vision/ROADMAP.md)

## Stages (in order)

| Order | File | Purpose |
|-------|------|---------|
| 1 | [audience.md](./audience.md) | Who the platform is for, segments, constraints, and open questions. |
| 2 | [brainstorming.md](./brainstorming.md) | Raw exploration: options, analogies, risks, and everything worth keeping before you commit to one direction. |
| 3 | [idea.md](./idea.md) | The **solid IDEA**: a concise, decision-ready description once brainstorming converges. |
| 4 | [how-to-use.md](./how-to-use.md) | How you want people to **use** it: journeys, workflows, terminology, and success moments. |

## Rules for this phase

- **Complete record**: Capture both your notes and assistant suggestions; label who said what where it matters (see section headers in each file).
- **Audience first**: Decisions in later files should trace back to audience choices (or explicit “deferred” notes).
- **Idea is a gate**: Do not treat `idea.md` as final until you are willing to describe the product in one paragraph to a stranger.
- **How-to-use last among these four**: It turns the idea into behavior and language users will recognize.
